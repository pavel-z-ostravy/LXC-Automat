      Ruční režim — Generovat setup.sh

📋 Jak postupovat

      
    
        
  1. Vytvoř nový LXC v Proxmox webUI — Ubuntu 22.04, privileged, 2 CPU, 2 GB RAM
    2. 
        
  2. 
          
            Uprav /etc/pve/lxc/[CT_ID].conf na Proxmox hostu
            ▶
          
          
    
            SSH na Proxmox a uprav conf soubor:
            
    ssh root@10.0.1.9
nano /etc/pve/lxc/202.conf
    
            Na konec souboru přidej tyto tři řádky (nutné pro Docker v LXC):
            
    lxc.apparmor.profile: unconfined
lxc.cgroup2.devices.allow: a
lxc.cap.drop:
    
            Ulož Ctrl+O → Enter → Ctrl+X, pak restartuj LXC:
            
    pct reboot 202
    
            
    Bez těchto řádků Docker v LXC kontejneru nefunguje.
    
          
    
        
    3. 
        
  3. 
          
            Nahraj setup.sh do LXC a spusť ho
            ▶
          
          
    
            Možnost A — pct push z Proxmox hostu (doporučeno) — klikni "☁ uložit na server", pak na Proxmox hostu (ssh root@10.0.1.9):
            
    pct push 202 /tmp/setup-lxc.sh /root/setup.sh
pct exec 202 -- bash /root/setup.sh
    
            
    Příkazy pct patří na Proxmox host — nespouštěj je uvnitř kontejneru.
    
            Možnost B — přes konzoli LXC (alternativa):
            
    pct enter 202
# uvnitř kontejneru:
bash <(cat << 'EOF'
# ... vlož celý obsah vygenerovaného setup.sh ...
EOF
)
    
            Možnost C — scp + SSH (jen pokud SSH už funguje):
            
    scp ~/setup.sh root@10.0.1.X:/root/
ssh root@10.0.1.X
bash setup.sh
    
            
    Skript sám vypíše SSH klíč k přidání na GitHub — počkej na konec výstupu.
