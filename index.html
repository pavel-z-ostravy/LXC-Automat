// Wizard log Starting... 

  


  

    
      Ruční režim — Generovat setup.sh
      ►
    
  

  

    

      
📋 Jak postupovat

      
    
        
  1. Create a new LXC in Proxmox web UI — Ubuntu 22.04, privileged, 2 CPU, 2 GB RAM
    2. 
        
  2. 
          
            Edit /etc/pve/lxc/[CT_ID].conf on the Proxmox host
            ▶
          
          
    
            SSH into Proxmox and edit the conf file:
            
    ssh root@10.0.1.9
nano /etc/pve/lxc/202.conf
    
            Append these three lines at the end (required for Docker inside LXC):
            
    lxc.apparmor.profile: unconfined
lxc.cgroup2.devices.allow: a
lxc.cap.drop:
    
            Save Ctrl+O → Enter → Ctrl+X, then reboot the LXC:
            
    pct reboot 202
    
            
    Without these lines Docker will not work inside the LXC container.
    
          
    
        
    3. 
        
  3. 
          
            Upload setup.sh into the LXC and run it
            ▶
          
          
    
            Option A — pct push from the Proxmox host (recommended) — click "☁ save to server", then on the Proxmox host:
            
    pct push 202 /tmp/setup-lxc.sh /root/setup.sh
pct exec 202 -- bash /root/setup.sh
    
            
    pct commands run on the Proxmox host — do not run them inside the container.
    
            Option B — via LXC console (alternative):
            
    pct enter 202
# inside the container:
bash <(cat << 'EOF'
# ... paste the full contents of the generated setup.sh ...
EOF
)
    
            Option C — scp + SSH (only if SSH is already working):
            
    scp ~/setup.sh root@10.0.1.X:/root/
ssh root@10.0.1.X
bash setup.sh
    
            
    The script will print the SSH public key to add to GitHub — wait for the output to finish.
